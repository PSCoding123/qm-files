text editor:  git config --global core.editor notepad

-contribute to a git project:
git fork: copy an exisisting project to your git hub account
git clone: clone an exisiting project to your pc
git status: shows changed files
git add . : add all files that have been changed (alternative git add -A)
git add <filename>: only adds the specified file
git commit: change the last commit
git checkout <file>: Revert a commit (by producing a new commit with contrary changes)
git pull <remote> <branch> publish local changes on a remote
git push : delete a branch on the remote


- create a repo on your computer:
make folder -> cd into it
copy files into folder
git init
git status: tells you the changes that have been made
git add "filename"
git restore --staged <file> : to unstage files
git commit -> file is now saved: test with delete and then
git checkout -- . (everything is now back to the way it was)

git remote -v: shows you the server adress of where it thinks you want to push to
git remote set-url origin https://github.com/your-user-name/your-project-name.git : sets the path to where git is supposed to push the files

git push origin <branch> : main, master whatever 

git add -i : interactivtly select files
	-> choose 4 (untracked)
	-> choose the files you want to select by number
	-> press empty
	-> quit
git commit -m "blabla"
